## Purpose / Big Picture

Resolve the currently open GitHub issues in one branch from `main`: stored-key
policy lookup must not fail on unnamed SQL expressions, operators must be able
to simulate internal service policy decisions by service name, routine Admin UI
success notices should clear themselves, and registered services need
configurable active health check endpoints.

## Surprises & Discoveries

- Observation: `stored_policy_for_key` was already fixed on `main`, but
  `policy_for_context` still selected unnamed `COALESCE(...)` columns and mapped
  decode failures to `control_state_unavailable`.
  Evidence: `crates/gateway-store/src/postgres.rs`.

## Decision Log

- Decision: Implement service health checks as additive `health_check_path` and
  `health_check_method` fields, preserving root-probe behavior when no path is
  configured.
  Rationale: This satisfies issue #55 without breaking existing services.
  Date/Author: 2026-05-25 / Codex.

- Decision: Use `service_name` as an additive policy simulator request field.
  Rationale: Existing callers stay compatible while the Admin UI can simulate
  `/services/*` policy against registered service names.
  Date/Author: 2026-05-25 / Codex.
