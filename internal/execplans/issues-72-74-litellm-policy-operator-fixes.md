## Context and Orientation

`crates/gateway-core/src/route_settings.rs` owns LiteLLM passthrough route
classification. Trusted-ingress bypass still requires passthrough enabled,
trusted-ingress UI exposure, explicitly exposed admin API exposure, method
allowlist match, and path allowlist match.

`crates/gateway-api/src/app.rs` owns the Admin API, including the policy
simulator response shape. `crates/gateway-api/admin-ui/src/main.ts` owns the
operator console source. Checked-in static assets under
`crates/gateway-api/src/static/admin-ui/` are generated by `npm run build:admin-ui`.

## Compatibility Boundary

Compatibility boundary: latest release tag `v0.1.12`. The change is additive:
existing route classifications and simulator fields remain available, no
PostgreSQL or Redis formats change, and the LiteLLM credential format default
remains `raw` for compatibility.
